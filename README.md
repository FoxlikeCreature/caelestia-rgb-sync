# caelestia-rgb-sync

Syncs all RGB lighting to the accent color generated by the [caelestia](https://github.com/caelestia-dots/shell) desktop shell. When you change your wallpaper, the LEDs follow within milliseconds.

Works with any hardware OpenRGB supports - motherboard, GPU, RAM, fans, keyboards, etc.

## How it works

caelestia derives an accent color from your wallpaper and writes it to a config file. This daemon watches that file with inotify and applies the color to all OpenRGB devices instantly via a persistent SDK connection.

Color pipeline per device (strict order):

```
wallpaper accent hex
  -> device.gain       (per-channel R/G/B multiplier)
  -> device.gamma      (per-channel gamma curve)
  -> device.black_point (per-channel minimum PWM floor)
  -> global.gain * global.brightness
  -> global.gamma
  -> color_temperature  (Kelvin correction, 6500 K = neutral)
  -> hue_corrections   (selective per-hue gain with linear falloff)
  -> LED color
```

All corrections are multiplicative. `black_point` is a minimum output floor applied after `gain` and `gamma` - it is not an additive offset.

## Requirements

- [OpenRGB](https://openrgb.org/)
- Python 3.11+
- [openrgb-python](https://github.com/jath03/openrgb-python)
- [tomli-w](https://github.com/hukkin/tomli-w) - for writing config from calibration TUI
- inotify-tools

Install on Arch/CachyOS:

```sh
yay -S openrgb python-openrgb python-tomli-w inotify-tools
```

> If your motherboard or RAM is not visible in OpenRGB, add `acpi_enforce_resources=lax` to your kernel parameters. This unlocks the SMBus that AMD FCH reserves for ACPI.

## Usage

```
caelestia-rgb-sync <command>

  daemon              watch scheme file, apply on change (run by systemd)
  apply               apply current color once and exit
  devices             list OpenRGB devices
  init                detect devices, write initial config
  calibrate device    per-device calibration - normalize all devices to one color
  calibrate global    global calibration - match LEDs to screen colors
  enable              enable systemd service
  disable             disable systemd service
```

## Quickstart

```sh
# 1. Detect devices and write initial config
caelestia-rgb-sync init

# 2. Calibrate devices (normalize all LEDs to the same color)
caelestia-rgb-sync calibrate device

# 3. Calibrate global settings (match LEDs to screen)
caelestia-rgb-sync calibrate global

# 4. Enable the daemon
caelestia-rgb-sync enable
```

## Configuration

`~/.config/caelestia-rgb/config.toml`

```toml
color_source = "primaryPaletteKeyColor"

# Per-device normalization - makes all devices show the same color.
# gain=[R,G,B]        : compensate for LED channel brightness differences
# gamma=[R,G,B]       : per-channel gamma (1.0 = linear)
# black_point=[R,G,B] : minimum PWM floor AFTER gain+gamma
# mode, force_mode, fast, skip : OpenRGB control flags

[devices."ENE DRAM"]
gain        = [1.0, 1.0, 1.0]
gamma       = [1.0, 1.0, 1.0]
black_point = [0.0, 0.0, 0.0]
mode        = "direct"

[devices."Radeon RX"]
gain        = [1.0, 1.0, 1.0]
gamma       = [1.0, 1.0, 1.0]
black_point = [0.0, 0.0, 0.0]
mode        = "direct"
fast        = false

# Global corrections applied after per-device normalization.
# brightness        : overall brightness multiplier
# color_temperature : Kelvin (6500 = neutral D65)
# gain=[R,G,B]      : global per-channel multipliers
# gamma             : global gamma exponent

[global]
brightness        = 1.0
color_temperature = 6500
gain              = [1.0, 1.0, 1.0]
gamma             = 1.0

# Hue corrections with linear falloff within hue_width.
# hue_center/hue_width : affected range on the color wheel (0.0-1.0)
# saturation_max       : use instead of hue to target white/desaturated colors
# gain=[R,G,B]         : multiplier at the center, rolls off to 1.0 at edges

[[global.hue_corrections]]
name       = "yellow-orange"
hue_center = 0.12
hue_width  = 0.15
gain       = [1.0, 0.7, 0.0]
```

### Device flags

| Key | Default | Description |
|-----|---------|-------------|
| `mode` | `"direct"` | OpenRGB mode name |
| `force_mode` | `false` | Always call set_mode before update |
| `fast` | `true` | Skip hardware ack (set false for some I2C GPU headers) |
| `skip` | `false` | Exclude this device |

## Calibration

LEDs have a very different color gamut from a monitor. Calibration brings them into alignment.

**Do per-device calibration first, global second.**

### Per-device calibration

Goal: make all devices show the same color when given the same input.

```sh
caelestia-rgb-sync calibrate device
```

The TUI cycles through R, G, B channels. Each step lights all devices in a single pure channel (red, then green, then blue). Adjust `gain`, `gamma`, and `black_point` for each device until brightness matches across all of them.

Controls:
- `w/s` or `Up/Down` - select device
- `a/d` or `Left/Right` - select field (gain / gamma / black_point)
- `+` / `-` - increase / decrease value
- `Tab` - cycle step size (0.05 / 0.01 / 0.001)
- `Enter` - confirm and move to next channel
- `q` - cancel

After all three channels, a rainbow sweep runs for visual verification. Press `Enter` to save or `q` to discard.

### Global calibration

Goal: match the overall color appearance of LEDs to what you see on screen.

```sh
caelestia-rgb-sync calibrate global
```

The TUI shows truecolor terminal swatches (red, green, blue, white, and midtone variants). Adjust `brightness`, `color_temperature`, `gain`, and `gamma` until the LEDs match the on-screen swatches.

Controls:
- `j/k` or `Up/Down` - select parameter
- `+` / `-` - increase / decrease value
- `Enter` - save
- `q` - cancel

After saving, a rainbow sweep runs for final verification.

## Troubleshooting

**Daemon not starting**

```sh
systemctl --user status caelestia-rgb.service
journalctl --user -u caelestia-rgb.service -n 50
```

**OpenRGB does not see my RAM or motherboard**

Add `acpi_enforce_resources=lax` to your kernel parameters.

**Device shows wrong color or ignores updates**

- Try `force_mode = true` for the device if it keeps switching modes
- Try `fast = false` if the color flickers or reverts
- Check that the `mode` name matches what `openrgb --list-devices` shows

**Colors apply but look very different from the wallpaper**

Run `caelestia-rgb-sync calibrate device` first, then `calibrate global`. Per-device normalization must come before global tuning.

## Using with a different shell

The daemon reads a hex color variable from a text file. Edit `SCHEME_PATH` and `color_source` in the config to point to your shell's equivalent.
