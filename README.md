# caelestia-rgb-sync

Syncs all RGB lighting in your system to the accent color generated by the [caelestia](https://github.com/caelestia-dots/shell) desktop shell. When you change your wallpaper, the LEDs follow within milliseconds.

Works with any hardware OpenRGB supports - motherboard, GPU, RAM, fans, keyboards, etc.

## How it works

caelestia derives an accent color from your wallpaper and writes it to a config file. This daemon watches that file with inotify and applies the color to all OpenRGB devices instantly. It holds a persistent connection to the OpenRGB SDK server so there is no startup overhead on each color change.

The color pipeline per device:

```
wallpaper accent hex
  -> global correction  (H/S/V multipliers + clamp, then R/G/B multipliers + clamp)
  -> per-device correction  (same, applied on top of global result)
  -> LED color
```

## Requirements

- [OpenRGB](https://openrgb.org/) - must support your hardware
- Python 3.11+
- [openrgb-python](https://github.com/jath03/openrgb-python) - the OpenRGB SDK client library
- inotify-tools - for filesystem watching

Install dependencies on Arch/CachyOS:

```sh
yay -S openrgb python-openrgb-python inotify-tools
```

> **Note:** If your motherboard or RAM is not visible in OpenRGB, you may need to add
> `acpi_enforce_resources=lax` to your kernel parameters. This unlocks the SMBus that
> AMD FCH reserves exclusively for ACPI. Without it, OpenRGB cannot access I2C-based
> devices like RAM and motherboard LED controllers.

## Installation

```sh
git clone https://github.com/FoxlikeCreature/caelestia-rgb-sync
cd caelestia-rgb-sync
bash install.sh
```

The installer:
1. Copies the script to `~/.local/bin/caelestia-rgb-sync`
2. Installs two systemd user units - one for OpenRGB, one for the sync daemon
3. Copies `config.example.toml` to `~/.config/caelestia-rgb/config.toml` if not present
4. Enables and starts both services

## Manual installation

If you prefer to set things up yourself:

```sh
# Script
install -Dm755 caelestia-rgb-sync ~/.local/bin/caelestia-rgb-sync

# Config
mkdir -p ~/.config/caelestia-rgb
cp config.example.toml ~/.config/caelestia-rgb/config.toml

# Systemd units
install -Dm644 systemd/openrgb.service ~/.config/systemd/user/openrgb.service
install -Dm644 systemd/caelestia-rgb.service ~/.config/systemd/user/caelestia-rgb.service

systemctl --user daemon-reload
systemctl --user enable --now openrgb.service
sleep 10
systemctl --user enable --now caelestia-rgb.service
```

## Configuration

`~/.config/caelestia-rgb/config.toml`

```toml
color_source = "primaryPaletteKeyColor"

# Global corrections - applied to every device.
# Each channel: scale (multiplier), floor (minimum), ceil (maximum).
# HSV channels are applied first, RGB channels on the result.
[global]
s = { scale = 1.55, floor = 0.55 }   # boost saturation; floor prevents white on pale colors
v = { floor = 0.70, ceil = 0.95 }    # brightness window

# Hue-range corrections - applied after global, selected by source color hue (0.0-1.0):
#   0.00 red    0.08 orange   0.17 yellow   0.33 green
#   0.50 cyan   0.67 blue     0.75 purple   0.83 magenta
[[hue_ranges]]
name = "yellow-orange"
from = 0.05
to   = 0.20
s    = { ceil = 0.55 }   # warm colors go oversaturated on LEDs

# Per-device corrections - applied last, on top of global + hue_range.
[[devices]]
match = "ENE DRAM"
mode  = "direct"
s     = { scale = 1.30 }
h     = { scale = 0.90 }
g     = { scale = 0.70 }   # ENE controllers have a strong green channel

[[devices]]
match = "Radeon RX"
mode       = "direct"
force_mode = true   # some I2C GPUs need set_mode before every update
fast       = false  # wait for hardware acknowledgment
```

### Channel reference

Each channel takes an inline table with up to three keys. All are optional - omit what you don't need.

`value = clamp(value * scale, floor, ceil)`

HSV channels are corrected first, then RGB channels on the result.

| Channel | Affects |
|---------|---------|
| `h` | Hue (0.0-1.0, full color wheel) |
| `s` | Saturation |
| `v` | Value (brightness) |
| `r` | Red |
| `g` | Green |
| `b` | Blue |

| Key | Default | Description |
|-----|---------|-------------|
| `scale` | `1.0` | Multiply the channel value |
| `floor` | `0.0` | Clamp minimum |
| `ceil` | `1.0` | Clamp maximum |

### Device-specific options

| Key | Default | Description |
|-----|---------|-------------|
| `mode` | `"direct"` | OpenRGB mode name to set |
| `force_mode` | `false` | Always call set_mode before updating color |
| `fast` | `true` | Skip waiting for hardware acknowledgment (faster, but some I2C devices need `false`) |
| `skip` | `false` | Exclude this device entirely |

### Applying config changes without rebooting

The daemon reloads the config on every color change. To trigger a reload without changing your wallpaper:

```sh
touch ~/.local/share/caelestia/hypr/scheme/current.conf
```

## Calibrating colors

LEDs never reproduce colors exactly as they appear on screen. Typical issues:

- **Washed out / pastel colors** - increase `s_mul` globally (try 1.3-2.0)
- **Too dark on some colors** - raise `v_min` (0.6-0.8 is a reasonable range)
- **Yellow/green cast on RAM** - lower `g_mul` for that device (try 0.7-0.85)
- **Wrong hue on a specific device** - adjust `h_mul` slightly below 1.0 to shift toward red, above 1.0 toward the other end

Start with global settings until the majority of your devices look right, then add per-device overrides for the outliers.

## Troubleshooting

**Daemon not starting**

```sh
systemctl --user status caelestia-rgb.service
journalctl --user -u caelestia-rgb.service -n 50
```

**OpenRGB does not see my RAM or motherboard**

Add `acpi_enforce_resources=lax` to your kernel parameters. On Limine, edit `/boot/efi/EFI/limine/limine.conf` and append it to your `CMDLINE` variable. On GRUB, add it to `GRUB_CMDLINE_LINUX_DEFAULT` in `/etc/default/grub` and run `grub-mkconfig`.

Reboot and check again - OpenRGB should now show the I2C devices.

**Device shows wrong color or ignores updates**

Some devices behave differently in the OpenRGB SDK:
- Try `force_mode = true` for the device
- Try `fast = false` if the color flickers or reverts
- Check that the `mode` name matches exactly what `openrgb --list-devices` shows

**Colors apply but look very different from the wallpaper**

This is normal - LED strips and fan LEDs have very different color gamuts from a monitor. Use the calibration steps above to tune per-device.

## Using with a different shell

The daemon watches `~/.local/share/caelestia/hypr/scheme/current.conf` and reads a hex color variable from it. If your shell writes accent colors somewhere else, edit the two constants near the top of the script:

```python
CONFIG_PATH = Path.home() / ".config/caelestia-rgb/config.toml"
SCHEME_PATH = Path.home() / ".local/share/caelestia/hypr/scheme/current.conf"
```

And set `color_source` in `config.toml` to the variable name as it appears in that file.
